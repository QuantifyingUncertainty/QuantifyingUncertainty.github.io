    1. Introduction

    The "Quantifying Uncertainty" project develops a code library, examples and documentation for the Generalized Metropolis-Hastings (GMH) algorithm detailed in (Calderhead, 2014). It can be used to run Monte-Carlo Markov Chain (MCMC) simulations of complex physical and biological processes to estimate process parameters. The GMH algorithm has been used in the scientific publications detailed below (TO DO: add publication list).

    The GMH MCMC code is written in Julia, a relatively new language designed for fast and flexible technical computing. It combines aspects of R, MATLAB and Python and can be easily parallelized to run on clusters and cloud computers. It has an active developer community making the language accessible from multiple platforms: Windows, Linux and MAC computers, and via cloud computing services such as Amazon Web Services or JuliaBox.

    Typical user situations of the code and documentation are:

    1. Reproduce the MCMC experiments that have been used in the scientific publications listed below. We recommend to run experiments from a pre-compiled machine image published on Amazon Web Services (AWS). The machine images contain all the required code, libraries and data to run the MCMC experiments for a particular paper, reducing your set-up time. By following the setup steps below, you will be able to re-run these experiments via a Jupyter/IJulia Notebook server, making them accessible via a web browser like Firefox or Chrome.
    2. Use the GMH code library to design your own MCMC experiments. The preferred way is to download the GMH repository from Github to your local machine or to work in the cloud via JuliaBox.

    The GMH repository can be obtained from http://uncertaintyquantification.github.io/GeneralizedMetropolisHastings.jl. It contains code to develop your own MCMC models. Documentation and examples included in the repository explain how. The rest of the documentation below outlines how to sign up for Amazon Web Services, how to start a pre-compiled AMI, how to set up the GMH repository on Juliabox or on your local machine, and how to set up and publish your own AMI to accompany the publication of scientific articles that use the GMH repository.

    3. AWS Setup and User Creation

    Step 3.1: Sign up for an AWS account

    This step requires a credit card to set up an account and a phone number to receive a security callback. By using Amazon's "Free Tier", it is possible to avoid being billed, but the credit card is still required.

    Please follow the instructions in the following AWS documentation:

    http://docs.aws.amazon.com/AWSEC2/latest/UserGuide/get-set-up-for-amazon-ec2.html

    You should complete the following steps outlined in the AWS documentation:

    1. Create an AWS account
    2. Create one or more IAM users
    3. Create a Security Group.

    Required Security Group Settings

    When setting up the Security Group, specify the following inbound ports:

    Service Port Source Access
    SSH 22 0.0.0.0/0 SSH command-line access
    Custom TCP Rule 8898 0.0.0.0/0 Web browser access to Julia server
    Custom TCP Rule 49152 - 65535 127.0.0.1/32 Internal ports for Julia server

    The “0.0.0.0/0” in the “Source” field means that incoming connections from any IP address are allowed. You can make the security group more secure by only allowing incoming connections from known IP addresses (e.g., "My IP" in the EC2 control panel). This means that every time you change your local IP address you will have to update the Security Group. As long as you are careful with the security steps outlined in Steps X and X below, the above settings should be fine.

    The entire process of setting up an account, creating IAM users and setting up a security group is likely to take between 1 and 2 hours, depending on your familiarity with AWS. These steps need to be completed only once.
